# Набор модулей для работы со счетами и картами клиентов.
PythonProject_Bank_ver_1

Задачи:
Маскировать/скрывать номера карт и счетов клиентов при заполнении
форм платежей и переводов.

Сортировать историю платежей по возрастанию и убыванию

1 -я функция:
get_mask_card_number
 принимает на вход номер карты и возвращает ее маску. 
Номер карты замаскирован и отображается в формате 
XXXX XX** **** XXXX
, где X — это цифра номера. 
То есть видны первые 6 цифр и последние 4 цифры, 
остальные символы отображаются звездочками, 
номер разбит по блокам по 4 цифры, разделенным пробелами. 
Пример работы функции:
7000792289606361     # входной аргумент
7000 79** **** 6361  # выход функции

2-я функция:
get_mask_account
 принимает на вход номер счета и возвращает его маску. 
Номер счета замаскирован и отображается в формате 
**XXXX, где X — это цифра номера.
То есть видны только последние 4 цифры номера,
а перед ними — две звездочки. Пример работы функции:
73654108430135874305  # входной аргумент
**4305  # выход функции

3-я функция:
filter_by_state
принимает список словарей и опционально значение для ключа 
state (по умолчанию 'EXECUTED'). Функция возвращает новый список словарей,
содержащий только те словари, у которых ключ 
state соответствует указанному значению.

4-я функция:
sort_by_date
принимает список словарей и необязательный параметр,
задающий порядок сортировки (по умолчанию — убывание).
Функция должна возвращать новый список, 
отсортированный по дате (date)

Модуль generators
Этот модуль будет содержит новые функции, реализующие 
генераторы для обработки данных.
1-я функция:
filter_by_currency
Принимает список словарей транзакций 
возвращает список отфильтрованный по коду валюты

2-я функция:
transaction_descriptions
Принимает список словарей транзакций 
возвращает действие со счетом

3-я функция:
card_number_generator 
Генерирует номер банковской карты в заданном диапазоне

## Тестирование
### Для тестирования вам потребуется:
установить интерпретатор Pycharm

установить фреймворк pytest  командой poetry add --group dev pytest

Настроить pytest из меню по пути:

current file -> edit configurations -> '+' -> add new configuration -> pytest -> 
script path -> указать директорию тестов и директорию проекта в целом.

Тесты проекта находятся в корне проекта в директории tests.
Conftest - директория хранения фикстур для тестов.
Test_masks - тесты проверки маскирования номера карты клиента и номера счета клиента.
Test_processing - тесты проверки фильтрации транзакций по выполнению и сортировке по дате.
Test_widget - тесты проверки на корректность получения даты транзакции и маски номера счета или карты.
для тестирования модуля generators
test_transaction_description
- Тест на выполение вывода типа операции со средствами клиента

test_filter_by_currency_with_usd_transaction
- Тесты на выполненние кода с заполнением и без

test_card_number_generator
тестирование согласно тестовым данным parametrize:
    заданные значения, пустой старт, пустой 'stop'

### Для проверки покрытия кода тестами вы можете использовать команду:
pytest --cov=src --cov-report=html

### Результирующий файл index.html проверки покрытия тестами кода,
Создается по вышеуказанной команде и храниться в директории htmlcov/index.html
